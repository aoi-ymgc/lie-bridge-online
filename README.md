# Lie Bridge Online

雲の上の一本橋をテーマにした、2〜4人用のオンライン対戦ブラフゲームです。  
React + TypeScript + Vite のクライアントと、Express + Socket.IO のサーバーで動きます。

## 起動方法

```bash
npm install
npm run dev
```

- クライアント: http://localhost:5173
- サーバー: http://localhost:3001

PowerShell で `npm` が実行ポリシーに止められる場合は、以下のように `npm.cmd` を使ってください。

```bash
npm.cmd install
npm.cmd run dev
```

## 公開方法

このゲームはリアルタイム通信に Socket.IO を使うため、GitHub Pages のような静的サイト公開だけでは動きません。  
公開する場合は Node.js サーバーを動かせるサービスへデプロイしてください。

おすすめ構成は、1つの Node.js アプリとして公開する方法です。

```bash
npm install
npm run build
npm start
```

本番では Express サーバーが `client/dist` を配信し、Socket.IO も同じURLで受けます。  
そのため、公開されたURLを知っている人はPCでもスマホでも同じURLから参加できます。

必要に応じて `CLIENT_ORIGIN` に許可するURLを指定できます。未指定の場合はテスト公開しやすいように全Originを許可します。

候補:

- Render
- Railway
- Fly.io
- Koyeb
- VPS

### Renderで無料公開する手順

1. このプロジェクトをGitHubリポジトリへアップロードします。
2. Renderにログインします。
3. `New` から `Web Service` を選びます。
4. GitHubリポジトリを接続します。
5. 以下の設定にします。

```txt
Runtime: Node
Build Command: npm install && npm run build
Start Command: npm start
Plan: Free
```

`render.yaml` も入れてあるため、RenderがBlueprintとして読み取れる場合はそのまま使えます。

無料枠では15分ほどアクセスがないとスリープします。  
次に誰かがURLを開いたとき、起動まで30〜60秒ほど待つことがあります。

GitHub Pages を使う場合は、フロントエンドだけをGitHub Pagesへ置き、Socket.IOサーバーは別サービスに置く必要があります。  
その場合はビルド時に `VITE_SERVER_URL` へサーバーURLを指定してください。

## 遊び方

1. プレイヤー名を入力してルームを作ります。
2. 表示された招待URLまたはルームIDを友達に共有します。
3. 2〜4人が参加したら、ホストがゲーム開始を押します。
4. 手番プレイヤーはダイスを振り、自分だけが出目を確認します。
5. 1〜4の数字を宣言します。出目と違う数字を言っても構いません。
6. 他プレイヤーは順番に30秒ずつ「ウソだ！」か「スキップ」を選びます。
7. 嘘を見破られたら手番プレイヤーのコマが落下します。
8. 宣言が本当だった場合は、疑ったプレイヤーのコマが落下します。
9. 嘘を見破ったプレイヤーは、宣言数字分だけ自分のコマを進めます。
10. 動けるコマがなくても、ゴール済みコマがあればそれを賭けて指摘できます。外すと賭けたゴール済みコマを失います。
11. 先に3個のコマをゴールさせたプレイヤーが勝利です。

## 操作説明

- ダイスを振る: 手番プレイヤーだけ押せます。
- 宣言 1 / 2 / 3 / 4: ダイス後、手番プレイヤーだけ押せます。
- ウソだ！: 宣言後、自分の順番が来たプレイヤーだけ押せます。active コマ、または賭けられるゴール済みコマが必要です。
- スキップ: 指摘しない場合に押します。全員がスキップすると手番の移動が確定します。
- もう一度遊ぶ: ゲーム終了後、ホストだけ押せます。
- ロビーに戻る: ゲーム終了後、ホストだけ押せます。

## 実装メモ

- ダイス結果はサーバー側で生成します。
- ダイス結果は手番プレイヤーにだけ `privateDiceResult` で送ります。
- コマ移動、落下、ゴール、勝敗判定はサーバー側で処理します。
- クライアントから送られる移動結果や出目は信用しません。
- 宣言後の指摘判断は1人30秒です。
- 宣言は30秒で時間切れになり、サーバーがランダムに自動宣言します。

## 動作確認方法

1. `npm run dev` を起動します。
2. http://localhost:5173 を2つ以上のブラウザタブで開きます。
3. 1つ目のタブでルームを作ります。
4. 2つ目以降のタブで招待URLまたはルームIDから参加します。
5. ホストでゲーム開始を押します。
6. 手番タブだけがダイスを振れることを確認します。
7. 出目が手番タブだけに表示されることを確認します。
8. 宣言後、他タブで「ウソだ！」を押し、出目公開と落下処理を確認します。
9. 3個ゴール、または全員が動けなくなるまで進め、終了画面を確認します。
10. ホストで再戦できることを確認します。
